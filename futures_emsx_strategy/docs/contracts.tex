\section{Overview}

\subsection{Architectural contract}

The system is partitioned so each layer depends only on the layer above it on the event flow, never the other way:

\begin{lstlisting}
Bloomberg market data
    -->  Market Data Service     (ticks, bars)
    -->  Strategy                 (TargetPosition)
    -->  Order Manager            (OrderIntent)
    -->  Pre-Trade Risk Gateway   (RiskDecision)
    -->  Execution Adapter        (ExecutionAck)
    -->  EMSX                     (route updates, fills)
    -->  Portfolio & Reconciler   (PositionBook, FillLedger, PnL)
\end{lstlisting}

\textbf{Hard rules:}
\begin{itemize}[leftmargin=*,itemsep=2pt]
  \item Strategies emit only \Cls{TargetPosition}. They never construct orders, never call EMSX, never know about routes.
  \item Order intents pass through \Cls{PreTradeRiskGateway} before reaching any \Cls{ExecutionAdapter}.
  \item Execution adapters are the only modules that import \code{blpapi}. Other layers work without it.
  \item Every \Cls{OrderIntent} has a deterministic \code{idempotency_key} (SHA-1 of strategy + instrument + bar-end + side + qty), so replays and restarts cannot duplicate orders.
\end{itemize}

\subsection{Event types (canonical)}

\begin{tabular}{@{}lll@{}}
\toprule
Event & Producer & Consumer(s) \\
\midrule
\code{MarketTick}      & Market data service             & Bar builder, tick store, stale monitor \\
\code{BarClosed}       & Bar builder                     & Strategy \\
\code{TargetPosition}  & Strategy                        & Order manager \\
\code{OrderIntent}     & Order manager                   & Risk gateway \\
\code{RiskDecision}    & Risk gateway                    & Execution service \\
\code{ExecutionAck}    & Execution adapter (sync)        & Order lifecycle \\
\code{ExecutionUpdate} & Execution adapter (async subs)  & Lifecycle, working book, monitoring \\
\code{FillUpdate}      & Execution adapter (async subs)  & Position book, PnL, fill ledger \\
\code{KillSwitchEvent} & Kill switch                     & All services (refuse to trade) \\
\bottomrule
\end{tabular}

\subsection{Notation used in this document}

Each entry contains:

\begin{description}[leftmargin=6em]
  \item[Purpose]   one-line summary.
  \item[Signature] full Python signature.
  \item[Pre]       caller-side preconditions.
  \item[Post]      observable effects after a successful call.

\end{description}

When an item has no preconditions worth stating, the \textbf{Pre} field is omitted.

\begin{contractbox}{\Cls{Clock} (abstract)}

\item[Purpose]   Time source. Decouples wall-clock from simulation/replay.

\begin{description}
\item[Purpose]   Wall-clock implementation.
\item[Signature] \lstinline|class SystemClock(Clock): def now(self) -> datetime|

\end{description}
\end{contractbox}

\begin{contractbox}{\Cls{FixedClock}}
\begin{description}

\item[Signature] \lstinline|class FixedClock(Clock): __init__(fixed: datetime) ; set(t: datetime) -> None|

\section{Module \texttt{market\_data/}}

\subsection{\texttt{market\_data.base}}

\begin{contractbox}{\Cls{MarketDataProvider} (abstract)}
\begin{description}
\item[Purpose] The single seam that decouples the strategy from Bloomberg. All implementations conform to this interface.
\item[Signature] \begin{lstlisting}
class MarketDataProvider(ABC):
    def start(self) -> None
    def stop(self) -> None
    def subscribe(self, instruments: list[str], fields: list[str]) -> None
    def on_tick(self, callback: Callable[[MarketTick], None]) -> None
    def request_historical_bars(self, instrument, start, end, interval_minutes=1) -> Iterable[BarClosed]
\end{lstlisting}
\item[Contract]
\begin{itemize}[leftmargin=*,itemsep=2pt]
  \item \code{start} initializes the session. Must be called before \code{subscribe}.
  \item \code{subscribe} is idempotent: re-subscribing the same instrument has no effect.
  \item \code{on_tick} callbacks are invoked once per \code{MarketTick} in arrival order; exceptions in callbacks must not kill the provider thread.
  \item \code{stop} releases the session; further calls are no-ops.
\end{itemize}
\end{description}
\end{contractbox}

\subsection{\texttt{market\_data.bloomberg\_provider}}

\begin{contractbox}{\Cls{BloombergMarketDataProvider}}
\begin{description}
\item[Purpose] BLPAPI implementation of \Cls{MarketDataProvider}. Threaded event loop pulls events from the session and dispatches \code{SUBSCRIPTION_DATA} events to callbacks.
\item[Signature] \begin{lstlisting}
__init__(host="localhost", port=8194,
         service="//blp/mktdata",
         historical_service="//blp/refdata",
         topic_to_symbol: dict[str, str] | None = None)
\end{lstlisting}
\item[\code{topic_to_symbol}] Optional translation map applied on receive: ticks subscribed under a Bloomberg topic (e.g. \code{ESM6 Index}) are reported under the application's internal symbol (e.g. \code{ES1}). When omitted, the topic is used verbatim.
\item[Wiring] \Cls{app.run_market_data} constructs the provider from \code{market_data.yaml} (host, port, service, historical\_service) and subscribes to \code{instruments.yaml::bloomberg_topic}; the translation map is built from \code{\{i.bloomberg_topic: i.symbol\}}.
\item[Pre]    \code{blpapi} is installed; reachable Bloomberg session (Desktop API, SAPI, or B-PIPE).
\item[Raises] \Cls{MarketDataError} if session start or service open fails, or if \code{blpapi} is missing.
\item[Fields supported] \code{BID, ASK, LAST_PRICE, VOLUME, EVT_TIME_STAMP} (any subset).
\end{description}
\end{contractbox}

\subsection{\texttt{market\_data.mock\_provider}}

\subsection{\texttt{market\_data.bar\_builder}}

\begin{contractbox}{\Cls{MinuteBarBuilder}}
\begin{description}
\item[Purpose] Streaming OHLCV aggregator. Closes the current bar deterministically when a tick crosses into the next minute (or \code{flush()} is called).
\item[Signature] \begin{lstlisting}
__init__(interval_minutes: int = 1)
on_tick(tick: MarketTick) -> None
on_bar(callback: Callable[[BarClosed], None]) -> None
flush(instrument: str | None = None) -> None
\end{lstlisting}
\item[Contract]
\begin{itemize}[leftmargin=*,itemsep=2pt]
  \item \code{on_tick} uses \code{exchange_timestamp} when present, else \code{receive_timestamp}.
  \item Price source order: \code{last} $\to$ mid of \code{(bid,ask)}; tick is dropped if neither is available.
  \item Bars emitted only when a strictly later minute arrives. Use \code{flush} at shutdown to emit pending bars.
\end{itemize}
\item[Raises] \code{ValueError} if \code{interval_minutes <= 0}.
\end{description}
\end{contractbox}

\subsection{\texttt{strategy.base}}

\begin{contractbox}{\Cls{PortfolioView} (Protocol)}
\begin{description}

class PortfolioView(Protocol):

\begin{contractbox}{\Cls{Strategy} (abstract)}
\begin{description}
\item[Purpose] Consumes bars, emits target positions. Pure logic; no I/O.
\item[Signature] \begin{lstlisting}
class Strategy(ABC):
    strategy_id: str

\end{lstlisting}

\subsection{\texttt{strategy.indicators}}

\begin{contractbox}{\Cls{SMA}, \Cls{EMA}, \Cls{RollingZ}}
\begin{description}

\item[\Cls{EMA}] Same shape; smoothing factor $\alpha = 2/(period+1)$.
\item[\Cls{RollingZ}] \code{__init__(period >= 2)}; returns \code{0.0} when sample stdev is zero, else $(x - \mu)/\sigma$.
\item[Raises] \code{ValueError} for non-positive periods.
\end{description}
\end{contractbox}

\subsection{\texttt{portfolio.positions}}

\begin{contractbox}{\Cls{PositionBook}}
\begin{description}
\item[Purpose] Authoritative in-memory net position + weighted-average cost per instrument. Updated by fills.
\item[Signature] \begin{lstlisting}

avg_cost(instrument: str) -> float | None
apply_fill(fill: FillUpdate) -> None

\end{lstlisting}
\item[Avg-cost rule] Let $q_{\text{prior}}$ be the position before the fill, $q'$ the signed fill quantity, and $q_{\text{new}} = q_{\text{prior}} + q'$.
\begin{itemize}[leftmargin=*,itemsep=2pt]
  \item \textbf{Open from flat / stack same side}: avg becomes the qty-weighted average of prior basis and fill price.
  \item \textbf{Partial close} ($q_{\text{prior}}$ and $q_{\text{new}}$ same non-zero sign): avg \emph{unchanged}.
  \item \textbf{Full close} ($q_{\text{new}} = 0$): avg reset to 0.
  \item \textbf{Flip} ($q_{\text{prior}}$ and $q_{\text{new}}$ opposite non-zero signs): new avg = fill price.
\end{itemize}
\item[Concurrency] Thread-safe (internal lock).
\end{description}

\subsection{\texttt{orders.order\_manager}}

\begin{contractbox}{\Cls{OrderManager}}
\begin{description}
\item[Purpose] Pure deduplicating mapping \Cls{TargetPosition} $\to$ list\ of \Cls{OrderIntent}.
\item[Signature] \begin{lstlisting}

         idempotency: IdempotencyStore,
         default_order_type: OrderType = OrderType.MKT,
         default_tif: TimeInForce = TimeInForce.DAY,
         max_clip: int | None = None)

on_target(target: TargetPosition) -> list[OrderIntent]
\end{lstlisting}
\item[Algorithm]
\begin{enumerate}[leftmargin=*,itemsep=2pt]
  \item $\Delta = \text{target.target\_qty} - \text{position} - \text{net working qty}$.
  \item If $\Delta = 0$, return \code{[]}.
  \item Otherwise \code{side = Side.from_delta(}$\Delta$\code{)}, \code{qty = |}$\Delta$\code{|}.

\end{enumerate}
\item[Why seen, not claim?] A transient risk rejection (stale data, session closed) would otherwise permanently poison the key and block retries.
\end{description}
\end{contractbox}

\begin{contractbox}{\Cls{EventBus} (abstract)}

\end{lstlisting}
\item[\code{Handler}] \code{Callable[[topic: str, payload: Any], None]}.
\item[Payload contract] Handlers always receive the canonical event dataclass registered in \Cls{messaging.codec} for that topic, never a raw dict --- including for serializing buses.
\end{description}
\end{contractbox}

\subsection{\texttt{messaging.codec}}

\begin{contractbox}{Topic registry + dataclass codec}
\begin{description}
\item[Purpose] Bridges JSON-serializing buses (Kafka, Redis) and typed handlers. Reconstructs the registered dataclass for each topic on decode so strategies and runners never have to branch on \code{dict} vs. dataclass.
\item[Signature] \begin{lstlisting}
register_topic(topic: str, event_type: type) -> None
topic_type(topic: str) -> type | None
encode(payload: Any) -> str
decode(topic: str, raw: Any) -> Any
to_jsonable(o: Any) -> Any
from_dict(cls: type, data: dict) -> Any
\end{lstlisting}
\item[Coercion rules] Enums via \code{cls(value)}; \code{datetime} via \code{fromisoformat}; nested dataclasses recurse; \code{Optional}/\code{|None} unions strip \code{NoneType}; lists/tuples recurse into their element type.
\item[Registration] \code{app.topics} performs the canonical bindings at import: \code{TICKS$\to$MarketTick}, \code{BARS$\to$BarClosed}, \code{TARGETS$\to$TargetPosition}, \code{INTENTS$\to$OrderIntent}, \code{RISK\_DECISIONS$\to$RiskDecision}, \code{EXECUTION\_UPDATES$\to$ExecutionUpdate}, \code{FILLS$\to$FillUpdate}, \code{KILL\_SWITCH$\to$KillSwitchEvent}. \code{register_topic} is idempotent; rebinding a topic to a different class raises \code{ValueError}.
\item[Unregistered topics] \code{decode} returns the raw value unchanged --- safe pass-through for custom topics.
\end{description}
\end{contractbox}

\subsection{\texttt{messaging.in\_memory\_bus}}

\subsection{\texttt{app.run\_execution}}

\begin{contractbox}{Entry point: \code{run\_execution}}
\begin{description}
\item[CLI] \code{... --config-dir config [--emsx|--paper] [--auto-route|--stage-only] [--metrics-port N]}.
\item[Subscriptions] \code{TICKS} (feed \Cls{StaleDataMonitor} and \Cls{InMemoryTickStore}) and \code{TARGETS}. The tick subscription is mandatory --- without it the risk gate rejects every order for stale data in split deployments.
\item[Pipeline (per \code{TARGETS} message)]
\begin{enumerate}[leftmargin=*,itemsep=2pt]
  \item \Cls{OrderManager}\code{.on_target} $\to$ intents.
  \item Publish \code{INTENTS}; run \Cls{PreTradeRiskGateway}\code{.validate}; persist via \Cls{RiskDecisionRepository}; publish \code{RISK_DECISIONS}.
  \item If approved: \code{idempotency.claim(key)}. Skip if it returns \code{False} (already submitted, e.g. duplicate target on the bus).
  \item Pre-register the order in \Cls{WorkingOrderBook} and \Cls{OrderLifecycle} under \code{intent.idempotency_key} (status \code{NEW}). \emph{Then} call \code{execution.submit_order}.
  \item Use \Cls{OrderLifecycle}\code{.set_venue_info} to record \code{ack.order_id} / \code{ack.route_id}. Do not overwrite status from the ack --- callbacks own it.
  \item On rejection ack: write \code{REJECTED} status to lifecycle and zero out the working entry.
  \item Inbound \code{ExecutionUpdate}/\code{FillUpdate} use \Cls{OrderLifecycle}\code{.resolve} and update lifecycle, working book, positions, PnL, metrics; publish \code{EXECUTION_UPDATES}, \code{FILLS}; on fill, also write to \Cls{FillRepository} and upsert \Cls{PositionRepository}.
\end{enumerate}
\item[Why pre-register?] The paper adapter fires \code{SENT}/fill/\code{FILLED} callbacks synchronously inside \code{submit_order}. Without pre-registration, the runner's post-submit bookkeeping would race the callbacks and leave a phantom working entry stuck at \code{SENT}.
\item[Why claim post-risk, not in OrderManager?] A transient risk rejection (stale data, session) must remain retryable. Claiming after risk approves leaves the key un-committed when risk rejects, so the next bar / restart can succeed.
\end{description}
\end{contractbox}

\section{Topology \& guarantees}

\subsection{Service boundaries}

\begin{center}
\small
\begin{tabular}{@{}lll@{}}
\toprule
Service              & Subscribes to               & Publishes \\
\midrule
market-data          & (Bloomberg, OS)             & \code{TICKS}, \code{BARS} \\
strategy             & \code{TICKS}, \code{BARS}   & \code{TARGETS} \\
execution            & \code{TICKS}, \code{TARGETS}& \code{INTENTS}, \code{RISK_DECISIONS}, \code{EXECUTION_UPDATES}, \code{FILLS} \\
reconciler           & (filesystem, OS)            & alerts \\
monitoring           & (Prometheus scrape)         & --- \\
\bottomrule
\end{tabular}
\end{center}

\textbf{Tick subscription is mandatory for any service that runs the pre-trade risk gate.} Without it, \Cls{StaleDataMonitor} sees the instrument as never seen and rejects every intent for stale data.

\subsection{Idempotency \& at-least-once delivery}

The bus is treated as \emph{at-least-once}. Idempotency is enforced inside the \Cls{OrderManager} via \code{idempotency\_key}:

\begin{lstlisting}
key = SHA1(strategy_id | instrument | bar_end | side | qty)[:16]
\end{lstlisting}

If the same key has already been claimed by \Cls{IdempotencyStore}, the intent is suppressed. \code{build_services} wires the store to \Cls{SqliteIdempotencyBackend}, persisting every claimed key into the \code{submitted_intents} table; a process crash or bar replay after restart therefore cannot resubmit a previously accepted order. The same key is the internal \code{order_id} used by paper-adapter callbacks, and is stored in \code{working\_orders.idempotency\_key} (\code{UNIQUE}) for the lifecycle audit trail.

\subsection{State-machine invariants}

\Cls{EMSXStateMachine} only permits the transitions in \code{\_ALLOWED}. Notable rules:
\begin{itemize}[leftmargin=*,itemsep=2pt]
  \item Terminal states (\code{FILLED, CANCELLED, REJECTED, ROUTE_ERR}) are sinks. Any further status change triggers a violation callback.
  \item \code{NEW} cannot jump directly to \code{SENT}; it must first be \code{RISK_APPROVED}.
  \item \code{UNKNOWN} is a recovery state --- any transition back into known states is allowed, used after gap-fill on restart.
\end{itemize}

\subsection{Phased rollout flags}

\begin{tabular}{@{}lll@{}}
\toprule
Phase & Adapter                                        & Notes \\
\midrule
1     & \Cls{PaperExecutionAdapter}                    & No external wire; full audit \\
2     & \Cls{EMSXExecutionAdapter} \code{auto_route=False} & Stages orders; trader routes \\
3     & \Cls{EMSXExecutionAdapter} \code{auto_route=True}  & Tight \code{risk\_limits.yaml}, one symbol, small size \\
4     & Same as 3                                       & Production limits, reconciler + kill switch live \\
\bottomrule
\end{tabular}

\subsection{Error budget for live trading}

Any of the following must trip the kill switch (manually or via alert action):
\begin{itemize}[leftmargin=*,itemsep=2pt]
  \item Market data age $>$ \code{stale_data_seconds} (default 30) for any subscribed instrument.
  \item Sustained risk rejections (e.g., $>$ 1/s for 5 min).
  \item State-machine violations (\code{EMSXStateMachine.on\_violation}).
  \item Position breaks reported by \Cls{Reconciler}.
\end{itemize}

\section{Quick reference --- minimum contracts to satisfy when extending}

\subsection{Adding a market-data vendor}

Implement \Cls{MarketDataProvider} (see \texttt{market\_data.base}). Bind it into \Fn{build_services} via the \code{market\_data} argument. No other layer changes.

\subsection{Adding a venue}

Implement \Cls{ExecutionAdapter}. Provide a venue-specific mapper analogous to \Cls{EMSXMapper}. Reuse \Cls{EMSXStateMachine} or write a venue-specific one. Bind via the \code{execution} argument to \Fn{build_services}.

\subsection{Adding a strategy}

Subclass \Cls{Strategy}. Read only via \Cls{PortfolioView}. Emit \Cls{TargetPosition}. Add an entry under \code{strategies.yaml}; the wiring layer instantiates it.

\subsection{Adding a risk check}

Add a private method \code{_check_*} to \Cls{PreTradeRiskGateway} that returns a \Cls{ValidationCheck}. Append it to the list inside \code{validate}. Tests must show that a failing case rejects the order with that check's name in \code{reasons}.

\subsection{Adding a bus implementation}

Implement \Cls{EventBus}. Register it in \Fn{make_bus}. The new transport must:
\begin{itemize}[leftmargin=*,itemsep=2pt]
  \item Serialize dataclasses correctly (see \Cls{KafkaBus} pattern).
  \item Catch and log handler exceptions.
  \item Provide at-least-once delivery (so \Cls{IdempotencyStore} can dedupe).
\end{itemize}
